# Suricata

The combination of Wazuh (which watches what happens inside the system) and Suricata (which watches what passes through the network) is an extremely powerful combination. It transforms Kali into a complete defense system (SIEM + IDS).

Note: Be aware of resources: Suricata consumes significant memory and CPU when analyzing traffic. If your machine is slow, it may slow down further.

---

## Install Suricata (if not already present)

```Bash
sudo apt update
sudo apt install suricata
```

---

## Configure Suricata to Listen on the Hotspot

You need to tell Suricata to listen on the correct interface (probably `wlan0`), otherwise it will listen on `eth0` (the cable) and won't see anything

Check the name of your Wi-Fi interface:

```Bash
ip a
```

(Look for the one with the `IP`, e.g. `wlan0`)

Modify the Suricata configuration:

```Bash
sudo nano /etc/suricata/suricata.yaml
```

Find the `af-packet` section: (scroll down a bit in the file). Change interface: `eth0` to the name of your interface (e.g. `wlan0`)

```YAML
# Example:
af-packet:
  - interface: wlan0  # <--- Change this
```

Update the detection rules (virus/attack signatures):

```Bash
sudo suricata-update
```

Start Suricata:

```Bash
sudo systemctl enable suricata
sudo systemctl start suricata
```

---

## Connect Wazuh to Suricata

Now we need to tell Wazuh: "Hey, read everything that Suricata writes and show it to me in the dashboard"

Suricata writes its logs to a file called `eve.json`. Wazuh just needs to read it

Open the Manager configuration file (since you are on the same machine):

```Bash
sudo nano /var/ossec/etc/ossec.conf
```

Scroll to the `<ossec_config>` section. Paste this code block at the end (but before the last line `</ossec_config>`):

```Bash
<localfile>
  <log_format>json</log_format>
  <location>/var/log/suricata/eve.json</location>
</localfile>
```

Save (CTRL+O, Enter) and exit (CTRL+X)

Restart the Wazuh Manager to apply the change:

```Bash
sudo systemctl restart wazuh-manager
```

## Let's Test if it Works

There is no point in configuring everything if we don't try to "trigger the alarm". Let's use a safe command that simulates an attack detected by Suricata

Open a terminal and run this command (generates a fake alert `"GPL ATTACK_RESPONSE id check returned root"`):

```Bash
curl http://testmyids.com
```

(You should see as response: `uid=0(root) gid=0(root) groups=0(root)`)

Check if Suricata detected it (look at the last lines of the log):

```Bash
tail -f /var/log/suricata/fast.log
```

You should see red text or a log entry related to `GPL ATTACK_RESPONSE`. (Press CTRL+C to exit)

---

## Check the Dashboard

Go to your Wazuh Dashboard (`https://localhost`)

Go to Wazuh -> Security events

In the search bar at the top, type: `rule.groups:suricata`

You should see the alert generated by your curl
