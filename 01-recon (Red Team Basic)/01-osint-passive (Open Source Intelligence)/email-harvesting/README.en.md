# OSINT Passive: Email Harvesting (Personal Audit)

> - **Phase:** Reconnaissance - Passive Information Gathering
> - **Visibility:** Zero - no direct contact with the target, querying public search engines
> - **Prerequisites:** Target domain, theHarvester tool installed, Internet access
> - **Output:** OSINT-002 - Map of exposed emails and attack surface analysis of the target domain

---

Objective: Verify the exposure of email addresses on the personal domain and analyze the attack surface of the digital portfolio.

Target: `nicholas-arcari.github.io`
Tools: `theHarvester`, `Google Dorks`

---

## 1 Theoretical Introduction

Email Harvesting is a reconnaissance technique typically used to map the organizational structure of a target organization.

However, in a Personal Security Audit context, this technique is adapted to verify whether a personal website inadvertently exposes contact data (PII) that could be indexed by search engines and subsequently targeted by spam bots or Spear Phishing campaigns.

---

## 2 Technical Execution

**Finding ID:** `OSINT-002` | **Severity:** `Informational`

#### A. Automated Scan (TheHarvester)

The portfolio domain was queried using `theHarvester` to aggregate results from public sources (OSINT) and verify the indexing of emails associated with the subdomain.

Command:

```Bash
theHarvester -d nicholas-arcari.github.io -b all
```

Technical Note: Since GitHub Pages is a static hosting service and does not provide a direct mail server, a null result for emails strictly tied to the domain (e.g., `info@nicholas-arcari.github.io`) is expected, unless there are erroneous custom configurations.

#### B. Results Analysis (TheHarvester)

Sources used: Baidu, Bing, DuckDuckGo, Yahoo, CRTsh.

|Category | Result |Technical Analysis|
|---------|-------------|-----------------|
|Exposed Emails | 0 email found | Positive. There is no direct exposure of email contacts indexed on search engines for this specific domain|
|Infrastructure | 4 IP (e.g., 185.199...) | The IP addresses belong to GitHub Pages infrastructure (Hosting Provider). They do not represent a private server managed by the user|
|Subdomains | 19 Hosts (e.g., shop., ftp.) | False Positives. This is an artifact due to DNS Wildcard records generated by GitHub's cloud infrastructure. Manual verification confirms these services are not active|

#### C. Targeted Search (Google Dorking)

Since the domain is hosted on GitHub, there is a risk that personal contact emails (e.g., Gmail) may have been left "in plaintext" in the source code or documentation files (README).

Query Executed:

```Bash
site:nicholas-arcari.github.io "gmail.com"
```

Objective: Identify whether the personal email address is visible in plaintext on portfolio pages.

The analysis revealed/did not reveal the presence of the personal address.

---

## 3 Conclusion and Remediation

The analysis shows that the personal domain presents a minimal attack surface regarding automated harvesting.

Recommendations:

- Obfuscation: If publishing a contact email is necessary, avoid the standard text format (`text@domain`). Use images or non-machine-readable formats (e.g., `name [at] gmail [dot] com`) to prevent scraping by bots.
- Contact Forms: Prefer the integration of backend-less forms (e.g., Formspree) that allow contact without exposing the destination address in the frontend code.
