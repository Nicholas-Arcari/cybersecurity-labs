I Providers (AWS / Azure)

Qui metti script e note specifiche per le piattaforme.

Focus Difensivo: Come configurare correttamente l'IAM (Identity and Access Management). Nel cloud, l'identità è il nuovo perimetro.

Focus Offensivo: Tecniche per enumerare permessi (es. "Posso leggere questo Bucket S3?"), Privilege Escalation su ruoli IAM mal configurati.

Nota: Il diagramma sopra (Shared Responsibility Model) è fondamentale per capire cosa devi proteggere tu (dati, accessi, config) e cosa protegge il provider (hardware, rete fisica).
